# 仕様書: モバイル今日ビュー強化

## 概要

モバイルの「今日」ビュー（カレンダーモード/カードモード共通）に以下を追加:
1. 全アイテムにチェックボックス（タスク + カレンダーイベント）
2. タップで編集モーダル（タスク名、時間、カレンダー種別）
3. タッチD&Dで時間変更（自動スクロール付き）

---

## 操作ルール

### チェックボックス

| 操作 | 対象 | 動作 |
|------|------|------|
| チェックON | タスク | `tasks.status` → `'done'`（既存PATCH API） |
| チェックON | カレンダーイベント | `event_completions` にレコード追加 |
| チェックOFF | タスク | `tasks.status` → `'todo'`（既存PATCH API） |
| チェックOFF | カレンダーイベント | `event_completions` からレコード削除 |
| 表示 | 完了済みアイテム | 取り消し線 + 薄いテキスト色、位置は変えない |

**制約**:
- チェックボックスのタップ領域は 44x44px 以上（モバイルタッチターゲット）
- チェック操作は optimistic update（即時UI反映、エラー時ロールバック）
- 終日イベントにもチェックボックスを表示する

### タップ編集モーダル

| 操作 | 動作 |
|------|------|
| アイテムをタップ | 編集モーダルを表示（D&D中は無効） |
| モーダル外タップ / 下スワイプ | モーダルを閉じる |
| 保存ボタン | 変更をAPIに送信、モーダルを閉じる |

**編集可能フィールド**:

| フィールド | タスク | カレンダーイベント |
|-----------|--------|------------------|
| タイトル | ✅ 編集可 | ✅ 編集可 |
| 開始時間 | ✅ 編集可 | ✅ 編集可 |
| 終了時間 | — （所要時間から自動計算） | ✅ 編集可 |
| 所要時間 | ✅ 編集可（15/30/60/90/120分） | — |
| カレンダー | ✅ 選択可（user_calendarsから） | 表示のみ（読み取り専用） |

**モーダルUI仕様**:
- 画面下部からスライドアップ（bottom sheet スタイル）
- 背景は半透明オーバーレイ
- ドラッグハンドル付き（上部中央の灰色バー）
- 閉じる: 下にスワイプ or オーバーレイタップ

### ドラッグ&ドロップ（時間変更）

| 操作 | 動作 |
|------|------|
| 長押し（350ms） | ドラッグモード開始、触覚フィードバック（vibrate） |
| ドラッグ中 | 半透明のゴーストを表示、元の位置にプレースホルダー |
| ドラッグ中 + 上下端付近 | 自動スクロール開始 |
| ドロップ | 15分刻みにスナップして時刻確定、API同期 |
| ドラッグキャンセル（元の位置に戻す） | 指を画面外に出す or 長押し中に離す |

**自動スクロール仕様**:

| 条件 | 動作 |
|------|------|
| 指が画面上端から60px以内 | 上方向にスクロール |
| 指が画面下端から60px以内 | 下方向にスクロール |
| 端からの距離0px | 最大速度（8px/frame） |
| 端からの距離60px | 速度0（停止） |
| 速度計算 | `speed = maxSpeed * (1 - distance / 60)` |

**D&Dビジュアルフィードバック**:
- ドラッグ中のアイテム: opacity 0.7、scale 1.02、shadow付き
- ドロップ先の時刻: 太字で「14:30」のようにプレビュー表示
- 元の位置: 点線ボーダーのプレースホルダー

**D&D後のAPI同期**:

| アイテム種別 | API | 同期先 |
|-------------|-----|--------|
| タスク | PATCH `/api/tasks/[id]` { scheduled_at } | Google Calendar（既存useTaskCalendarSync経由） |
| カレンダーイベント | PATCH Google Calendar API（start_time, end_time） | ローカルキャッシュ更新 |

**15分刻みスナップ**:
- ドロップ位置を最も近い15分に丸める
- 例: 14:07 → 14:00、14:08 → 14:15、14:22 → 14:15、14:23 → 14:30

## フロー

### チェックボックストグルフロー

```
ユーザーがチェックボックスをタップ
  ↓
アイテムの種類を判定
  ├─ タスク → PATCH /api/tasks/[id] { status: 'done'|'todo' }
  └─ カレンダーイベント
       ├─ チェックON → POST /api/event-completions { google_event_id, calendar_id }
       └─ チェックOFF → DELETE /api/event-completions/[google_event_id]
  ↓
UI即時更新（optimistic）
  ↓
API成功 → 完了
API失敗 → UI ロールバック + エラートースト
```

### タップ編集フロー

```
ユーザーがアイテムをタップ（チェックボックス以外の領域）
  ↓
ドラッグモード中？ → Yes: 無視
  ↓ No
編集モーダルを表示（bottom sheet）
  ↓
フィールドを編集
  ↓
保存ボタンをタップ
  ├─ タスク → PATCH /api/tasks/[id] { title, scheduled_at, estimated_time, calendar_id }
  │           → Google Calendar 同期（既存フロー）
  └─ イベント → Google Calendar API PATCH { summary, start, end }
               → ローカルキャッシュ更新
  ↓
モーダルを閉じる + タイムライン更新
```

### D&Dフロー

```
ユーザーがアイテムを長押し（350ms）
  ↓
触覚フィードバック（navigator.vibrate(50)）
  ↓
ドラッグモード開始
  ├─ ゴーストアイテムを指に追従させる
  ├─ 元の位置にプレースホルダー表示
  └─ チェックボックス・タップ編集を無効化
  ↓
ドラッグ中
  ├─ 指の位置から時刻を計算（Y座標 → 時:分）
  ├─ 15分刻みにスナップ
  ├─ 時刻プレビューを表示
  └─ 上下端付近 → 自動スクロール開始
  ↓
指を離す（touchend）
  ↓
新しい時刻でAPI更新
  ├─ タスク → PATCH /api/tasks/[id] { scheduled_at: 新時刻 }
  │           → Google Calendar 即時同期
  └─ イベント → Google Calendar API PATCH { start, end }
  ↓
タイムライン更新
```

---

## 制約

1. **D&Dとタップの区別**: 長押し350ms未満 = タップ（編集モーダル）、350ms以上 = D&D
2. **D&Dと通常スクロールの区別**: D&Dは長押しトリガーのため通常スクロールと干渉しない
3. **終日イベント**: D&D対象外（時間帯がないため）。チェックボックスとタップ編集は有効
4. **カレンダーモードのみ**: D&Dはカレンダーモード（タイムライン表示）でのみ有効。カードモードでは無効
5. **API同期**: D&D後のGoogle Calendar同期はベストエフォート。失敗してもローカルの時刻変更は維持
6. **event_completions保存期間**: 30日。古いレコードのクリーンアップはバッチまたは手動

## エッジケース

| ケース | 対応 |
|--------|------|
| D&D中に画面外に指が出る | ドラッグキャンセル、元の位置に戻す |
| D&D中にアプリがバックグラウンドへ | ドラッグキャンセル |
| 繰り返しイベントの時間変更 | 「この予定のみ」で変更（シリーズ変更はサポートしない） |
| 完了チェック後にイベントがGoogleから削除される | 次回同期時にevent_completionsは残るが表示されない（問題なし） |
| 複数日にまたがるイベント | D&D対象外、チェックボックスは有効 |
| 0分/短いイベント（5分未満） | D&Dのタッチ領域確保のため最小表示高さ30px |
| オフライン時のD&D | UIは更新、API送信はオンライン復帰時にリトライしない（エラートースト表示） |
| カードモードでの操作 | チェックボックスとタップ編集のみ。D&Dなし |
