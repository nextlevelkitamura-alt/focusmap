```text
Use case: ui-mockup
Asset type: mobile web/iOS settings overview mockup
Primary request: Design the mobile top-level settings screen for Focusmap.
Product context: Focusmap is mobile-first. Settings should be usable one-handed and should show whether AI execution, calendar, API access, and project execution targets are healthy.
Screen: /dashboard/settings mobile overview
Canvas: iPhone-sized portrait mockup, black/white/gray Focusmap UI.
Layout: mobile list-detail launcher. Top has title "設定", search field, then a compact operational status block. Below are grouped list rows with icons, status chips, subtitles, chevrons.
Rows:
- AI / 自動化: 接続中

- 連携: Google Calendar 接続中
- アクセス/API: 2 keys
- 外観: システム
Components: 44px+ rows, monochrome status chips, chevrons, no desktop sidebar, no dense tables on root, bottom account profile row with circular initial/avatar and logout/account menu affordance.
State requirements: show one attention chip and several normal chips. Do not include an "概要" row inside the mobile list; the page itself is already the overview/top.
Responsive note: this is mobile; detail pages drill in with back button.
Text: Japanese labels, short subtitles, no paragraphs.
Visual style: Focusmap mobile app, compact, high contrast black/white/gray, no marketing. Use smaller modern mobile settings typography and monochrome status labels.
Constraints: no decorative blobs, no fake logos, no emoji icons, no tiny text, no card-heavy landing page, no overlapping bottom nav.
```
