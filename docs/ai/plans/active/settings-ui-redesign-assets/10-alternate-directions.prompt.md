# 10 Alternate Directions Prompt

Use these only if Direction A needs revision after visual review.

## Direction B: Mobile-First Simple

```text
Use case: ui-mockup
Asset type: settings UI visual direction
Primary request: Create a mobile-first Focusmap settings system that prioritizes one-handed use and simple drill-in lists.
Product context: AI execution dashboard for a Mac/iPhone power user.
Layout thesis: mobile root list first, desktop is a widened list-detail version. Minimal status tiles, fewer desktop tables, more grouped rows.
Keep: status chips, AI/Mac/Calendar/API state on root, one job per screen.
Reject: dense admin tables, multi-column color managers, verbose diagnostic copy.
Visual style: compact native-app-like grouped settings, Focusmap dark neutral, restrained color.
Constraints: no decorative blobs, no marketing hero, no fake logos, no nested cards.
```

## Direction C: Dense Admin Console

```text
Use case: ui-mockup
Asset type: settings UI visual direction
Primary request: Create a dense admin/control-center direction for Focusmap settings.
Product context: AI execution control with local Mac agent, Codex, external API keys, Google Calendar, project repo paths.
Layout thesis: desktop-first, status tables and connection rows, left sidebar with search and badges. Mobile is supported but secondary.
Keep: high scan efficiency, API key list, repo path table, Google Calendar selected list.
Reject: playful mobile-native grouped cards, large previews.
Visual style: GitHub/Linear-like developer settings, restrained dark mode, precise status language.
Constraints: no decorative blobs, no fake logos, no tiny unreadable table text.
```

## Direction D: Conservative Productive

```text
Use case: ui-mockup
Asset type: settings UI visual direction
Primary request: Create a conservative settings redesign that changes IA and consistency without a dramatic visual shift.
Product context: Focusmap existing settings already has sidebar, list rows, and dark mode. Improve it carefully.
Layout thesis: keep current shell, rename/generalize categories, add status summary and shared settings rows, avoid big layout changes.
Keep: existing route structure, existing card/list proportions, existing lucide icons.
Reject: new visual system, heavy search, big table redesign.
Visual style: familiar Focusmap settings, better hierarchy and state clarity.
Constraints: no decorative blobs, no marketing hero, no nested cards, no major palette shift.
```
