# スペース/プロジェクト階層UIの再設計 — 仕様書

> **ステータス**: Approved（プラン承認済み・2026-05-21）
> **関連**: [project-context-simplification.md](project-context-simplification.md)
> **前提**: DBスキーマ変更なし・新エンティティ追加なし。既存 spaces/projects テーブルと CRUD API を流用。

---

## 1. 背景と課題

FocusMap は「スペース → プロジェクト → マインドマップ/メモ」の3階層。現状の階層UIに3つの問題:

1. **スペース作成がヘッダーのドロップダウンに埋もれている**（3ステップ・発見性が低い）。
2. **左サイドバーにスペースの概念が見えない** — プロジェクト一覧だけが並び、上位概念が不可視。
3. **スペース管理（ヘッダー）とプロジェクト管理（左サイドバー）でUIパターンがバラバラ** — 両者の
   関連が直感的に伝わらない。

加えて中央ペインに現在のプロジェクト表示がなく「今どこにいるか」が分かりにくい。

リサーチ（Notion / Linear / ClickUp / Google / Slack）の収束結論に基づき再設計する。

---

## 2. 設計方針

- **左サイドバーを「スペース → プロジェクト」の入れ子ツリーに統合**。スペースを見出しセクション化し、
  配下にプロジェクトをインデント表示（Notion teamspaces パターン）。
- **サイドバーは折りたたみ式**。作業中は畳んでキャンバスを最大化。状態は localStorage に永続化。
  デフォルトは畳んだ状態。
- **切り替えの主導線は中央上部のパンくず**。`スペース名 › プロジェクト名` をクリックすると
  ポップオーバーで切替（サイドバーを開かずに済む）。
- **UIパターンを統一** — スペースもプロジェクトも「行 ＋ 横の `+` ＋ ホバー操作」に揃える。
- **ヘッダーのスペースドロップダウンは廃止**（サイドバー＋パンくずに集約）。
- スペースに色＋アイコンを与え視覚識別。**アクセント1色のみ**。アプリ全体のテーマ着色はしない。
- モバイルはボトムシートの「スペース→プロジェクト」ツリー。
- 設定に「スペース/プロジェクト管理」画面（権限・削除・アーカイブ等の低頻度操作専用）。
  日常ナビはサイドバー/パンくずで完結させ、設定画面をナビの主役にしない。
- 階層は3つのまま増やさない（ClickUp の深い階層は「複雑すぎ」と批判される反面教師）。

---

## 3. データモデル

**スキーマ変更なし。** 既存のまま使う:
- `spaces`: `title` / `color` / `icon` / `default_calendar_id` 等（色・アイコンは既存カラム）。
- `projects`: `space_id`（親）/ `title` / `status` / `color_theme` / `description`（Part Aで追加済み）。
- CRUD API は実装済み:
  - スペース: `POST /api/spaces`, `PATCH/DELETE /api/spaces/[id]`
  - プロジェクト: `POST /api/projects`, `PATCH/DELETE /api/projects/[id]`

## 4. UI設計

### 4.1 通常（作業中）— サイドバーは折りたたみ、切替はパンくず

```
┌┐┌─ マインドマップ（最大）─────────────────┐
│▸││ 🟦 仕事 › 採用プロジェクト  ▾ ←クリック  │
│▸││ ──────────────────────────────────────│
│▸││            （マインドマップ）           │
└┘└─────────────────────────────────────────┘
 ↑細いバー（クリックでツリー展開）
```

パンくず `▾` クリック → 切替ポップオーバー:

```
  ┌─ 切り替え ───────────┐
  │ ▾ 🟦 仕事             │
  │     ● 採用プロジェクト ✓│
  │     ● 営業改善         │
  │ ▸ 🟩 プライベート       │
  │ ＋ スペース / プロジェクト│
  └──────────────────────┘
```

### 4.2 展開時 — じっくり一覧・管理する時だけ開く

```
┌─ サイドバー（展開）──────────┐
│  スペース            [ + ] │
│ ────────────────────────── │
│ ▾ 🟦 仕事                   │
│     ● 採用プロジェクト ◀選択 │
│     ● 営業改善              │
│     ＋ プロジェクトを追加     │
│ ▸ 🟩 プライベート            │
│ ▸ 🟪 学習                   │
└─────────────────────────────┘
```

- スペース行: chevron（▸/▾）＋ 色ドット/アイコン ＋ 名前。ホバーで編集/削除。
- プロジェクト行: インデント ＋ 色ドット ＋ 名前。選択中は左ボーダー。ホバーで操作メニュー。
- スペース見出し横 `[+]` でスペース作成、各スペース配下 `＋` でプロジェクト作成。

### 4.3 スペース作成モーダル（軽量）

```
┌─ 新しいスペース ──────────┐
│  名前   [ 仕事          ] │
│  色     🟦 🟩 🟥 🟪 🟧   │
│  アイコン 💼 🏠 📚 🎯 ⭐  │
│              [ 作成 ]    │
└──────────────────────────┘
```

色は `src/lib/color-utils.ts` の `COLOR_PRESETS` を流用。

### 4.4 モバイル — ボトムシートで切り替え

```
┌──────────────────────────┐
│  スペースを切り替え         │
│ ──────────────────────────│
│  🟦 仕事            ✓ 今ここ│
│  🟩 プライベート            │
│  🟪 学習                   │
│  ＋ 新しいスペース          │
└──────────────────────────┘
```

既存 `mobile-project-selector.tsx`（Popover）をボトムシート階層ツリーへ発展。

## 5. 実装ステップ（コンポーネント単位）

| Step | 内容 | 主なファイル |
|---|---|---|
| B-1 | 左サイドバーを折りたたみ可能な入れ子ツリーに。`Section`/`ProjectCard`/作成フォームを再編。畳む/開くトグル＋localStorage永続化（デフォルト畳む） | `src/components/dashboard/left-sidebar.tsx` |
| B-2 | スペース作成モーダル（名前＋色＋アイコン）。`POST /api/spaces` 利用 | `left-sidebar.tsx` 内 or 新規 `space-create-dialog.tsx` |
| B-3 | ヘッダーのスペースドロップダウン廃止 | `src/components/layout/header.tsx`（L156-345） |
| B-4 | 中央ペイン上部にクリック可能パンくず＋切替ポップオーバー（サイドバーを開かず切替する主導線） | `src/components/dashboard/center-pane.tsx`、新規 `space-project-switcher.tsx` |
| B-5 | モバイルをボトムシート階層ツリーに | `src/components/mobile/mobile-project-selector.tsx` |
| B-6 | 設定の「スペース/プロジェクト管理」画面（低頻度操作専用） | `src/app/dashboard/settings/spaces/` |

状態管理（`selectedSpaceId` 等）は `src/app/dashboard/dashboard-client.tsx`（L58 / L83-88 / L301-312）の
既存ロジックを流用。`color-utils.ts` の色定義を再利用。

## 6. リスクと対策

| リスク | 対策 |
|---|---|
| 過去に類似の「スペース＋左サイドバー再設計」が中止 | 新エンティティを足さず、既存テーブル/API流用・段階実装で再挑戦 |
| サイドバーが作業領域を圧迫 | 折りたたみ式＋パンくず切替で、作業中はキャンバス最大化 |
| 作り込みすぎ | 階層3つ固定・アクセント1色・設定画面はナビ主役にしない |
| 既存の `selectedSpaceId` 連携の退行 | 状態管理ロジックは流用し、表示層のみ差し替え |
| 作業ツリーに他作業のWIPが多数 | コミットは自分のファイルのみ選んで行う |
