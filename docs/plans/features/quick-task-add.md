# 設計プラン: クイックタスク追加 + サブタスク追加

## 要件

### 【A】クイックタスク追加（FAB）

今日ビューの右下にフローティング「+」ボタンを配置。
マインドマップを開かずに、タスクをサッと追加できる。

**操作フロー:**
1. FABボタンをタップ
2. ボトムシートが開く（モバイルネイティブな操作感）
3. 入力:
   - タスク名（必須）
   - プロジェクト選択（ドロップダウン、**任意**）
   - 開始時間（タップで時刻ピッカー、任意 → `scheduled_at`）
   - 見積もり時間（分単位、任意 → `estimated_time`）
   - カレンダー選択（ドロップダウン、任意 → `calendar_id`）
   - 優先度（1-5、デフォルト: 3）
4. 「追加」ボタンで保存

**プロジェクト選択あり → マインドマップに追加:**
- `project_id: 選択されたプロジェクト`
- `parent_task_id: null`（ルートレベル）
- マインドマップ画面でプロジェクト直下に表示
- 後からD&Dでグループに移動可能

**プロジェクト未選択 → 今日ビュー専用タスク:**
- `project_id: null`（マインドマップには表示されない）
- 今日ビューのタイムラインにのみ表示
- カレンダーイベントと同じ感覚で使える:
  - ☑ チェックボックス（完了/未完了）
  - ⏱ タイマー（時間記録）
  - 優先度・見積もり時間の表示

**即時反映（重要）:**
- 追加したタスクは以下に**即座に**反映させる:
  1. **スマホ今日ビュー**: タイムラインに即時表示
  2. **PC カレンダー（右サイドバー）**: 週ビュー/日ビューに即時表示
  3. **Google カレンダー同期**: `scheduled_at` + `estimated_time` + `calendar_id` が揃っていれば
     既存の `useTaskCalendarSync` でGoogleカレンダーにも即時同期
- ページリロード不要。楽観的更新で即座にUIに反映する

### 【B】サブタスク追加（カレンダー横+）

今日ビューのカレンダータイムライン上の各タスクの右端に
モダンでシンプルな「+」ボタンを配置。
タスクの中の細かい作業ステップをその場で追加できる。

**操作フロー:**
1. タスク横の「+」ボタンをタップ
2. インライン入力欄が表示される（軽量、ボトムシート不要）
3. タスク名を入力 → Enter で追加
4. 子タスクとして保存:
   - `parent_task_id: 親タスクのID`
   - `project_id: 親タスクと同じ`
5. 追加された子タスクは親タスクの下にチェックリストとして表示
6. 各子タスクにはチェックボックス + ミニタイマー

**子タスクの表示:**
- 親タスクの下にインデントされたチェックリスト形式
- チェックボックス: タップで status を todo ↔ done 切り替え
- ミニタイマー: タップで時間記録開始/停止
- 完了率バー: 親タスクの下に薄く表示（2/5 完了 など）

## リスク評価

| リスク | レベル | 対策 |
|--------|--------|------|
| 既存マインドマップとの同期 | LOW | 既存 useMindMapSync の仕組みをそのまま活用 |
| プロジェクト一覧の取得 | LOW | 既存の spaces/projects API を利用 |
| FABとスクロールの干渉 | LOW | position: fixed + z-index で対処 |
| 子タスクの表示がタイムラインを崩す | MEDIUM | 折りたたみ式で初期は非表示 |

## 依存関係

- **ライブラリ**: 新規追加なし
- **外部サービス**: なし
- **DB変更**: なし（既存 tasks テーブルで対応可能）
  - `parent_task_id: null` → ルートレベルタスク
  - `parent_task_id: <id>` → サブタスク
  - `project_id` → プロジェクト紐付け
  - `is_group: false` → 通常タスク
