## 1. 初期テンプレの選定基準

### 1.1 grill-meで合意した「汎用」の制約

- **業界特化しない** (CA業務専用テンプレは初期に入れない、北村の個人用Workspaceには別途置く)
- 個人 + 法人 両方で使えるもの
- 「AIに不慣れな社員」が触って「便利」と感じる即効性

### 1.2 採用基準

| 基準 | 重み |
|---|---|
| 認証が比較的取りやすい (Google / メール / LINE) | 高 |
| 結果が「見える」(レポート / サマリ / 通知) | 高 |
| 失敗してもダメージが小さい (read系優先) | 高 |
| 「これ毎朝便利」と感じる定期実行系 | 高 |
| Browser automation で実装可能 (DOM操作可) | 必須 |

### 1.3 却下した候補

| 却下 | 理由 |
|---|---|
| 経理処理・架電リスト更新 (CA業務由来) | 業界特化、初期には入れない |
| Salesforce/HubSpot連携 | API認証が複雑、Enterprise向けに後回し |
| 銀行口座のスクレイピング | 規約違反リスク、銀行毎の認証差異 |
| 領収書OCR | OCR精度の調整に時間がかかる、Phase 4以降 |

## 2. MVP 初期スキル 7個

### 2.1 一覧

| # | スキル名 | カテゴリ | approval_type | 認証 |
|---|---|---|---|---|
| 1 | 📧 **メール要約** | 朝の準備 | auto | Gmail / Outlook |
| 2 | 📅 **今日のカレンダー整理** | 朝の準備 | auto | Google Calendar |
| 3 | 💬 **未読メッセージ集約** | 朝の準備 | auto | LINE / Slack |
| 4 | 🌐 **競合・情報サイト巡回** | 情報収集 | auto | (認証不要) |
| 5 | 📝 **議事録要約** | 文書化 | confirm | (録音アップロード) |
| 6 | 📊 **問い合わせフォーム集約** | 営業/CS | confirm | Webサイト管理画面 |
| 7 | 🌅 **朝のブリーフィング** | 統合 | auto | 上記の1-3を統合 |

### 2.2 各スキルの詳細

#### スキル1: 📧 メール要約

```
やること:
  Gmail/Outlookの未読メールを開き、AIが3行で要約
  重要度判定 (要対応 / 通知のみ / スパム)
  Webアプリに一覧表示

ステップ:
  ① メールサービスにログイン (Cookie利用)
  ② 過去24時間の未読メールを取得 (最大50件)
  ③ 各メールをAIで要約 + 重要度判定
  ④ 結果をスキルカードに表示

使う頻度:    毎朝 (定期実行候補)
実行時間:    2-5分
1実行原価:   $0.05-0.10 (Gemini Flash)
```

#### スキル2: 📅 今日のカレンダー整理

```
やること:
  今日のカレンダー予定を取得
  空き時間を可視化
  「今日中にやるべき仕事」のオススメ時間帯を提案

ステップ:
  ① Google Calendar から今日の予定取得 (既存連携を利用)
  ② AIで「移動時間」「会議の準備時間」を加味
  ③ 空き時間とオススメ作業を提案

使う頻度:    毎朝 + 急な予定変更時
実行時間:    1-2分
1実行原価:   $0.02-0.05
```

#### スキル3: 💬 未読メッセージ集約

```
やること:
  LINE / Slack / Discord の未読を一覧化
  返信優先度をAI判定 ("今すぐ" / "今日中" / "後で")
  返信ドラフトを生成 (確認後送信)

ステップ:
  ① 各サービスにログイン
  ② 未読メッセージ取得
  ③ AIで優先度判定 + 返信ドラフト
  ④ Webアプリに表示 (承認後に送信)

使う頻度:    1日3-5回
実行時間:    2-3分
1実行原価:   $0.05-0.10
```

#### スキル4: 🌐 競合・情報サイト巡回

```
やること:
  事前登録したサイト5つを巡回
  新着記事・更新情報をサマリ
  特定キーワードを含む情報のみ抽出可

ステップ:
  ① ユーザーが登録したURL一覧を取得
  ② 各サイトにアクセス、新着取得
  ③ AIで要約 + キーワードフィルタ
  ④ Webアプリにダイジェスト表示

使う頻度:    毎朝 (定期実行候補)
実行時間:    5-10分
1実行原価:   $0.10-0.20
```

#### スキル5: 📝 議事録要約

```
やること:
  録音ファイル or 文字起こしテキストから議事録を生成
  「決定事項」「TODO」「議題未着手」を分類
  発言者別の発言量も集計

ステップ:
  ① ユーザーが録音/テキストをアップロード
  ② AIで文字起こし (録音時) → 構造化要約
  ③ 議事録テンプレに整形
  ④ Webアプリで確認、修正可

使う頻度:    会議毎
実行時間:    3-10分 (会議の長さに比例)
1実行原価:   $0.10-0.50 (会議1時間あたり)
```

#### スキル6: 📊 問い合わせフォーム集約

```
やること:
  自社Webサイトの問い合わせ管理画面にログイン
  新規問い合わせを取得
  カテゴリ分類 + 返信優先度判定
  スプシ/Notionに転記

ステップ:
  ① 管理画面にログイン (Cookie利用)
  ② 新規問い合わせ取得
  ③ AIでカテゴリ分類
  ④ 既存のスプシ/Notionに転記 (確認後)

使う頻度:    毎朝 + 緊急時
実行時間:    3-5分
1実行原価:   $0.05-0.15
```

#### スキル7: 🌅 朝のブリーフィング

```
やること:
  スキル1〜3 を統合した「朝の1画面」
  メール要約 + カレンダー + 未読メッセージ
  + 「今日の優先事項」をAIが提案

ステップ:
  ① スキル1, 2, 3 を並列実行
  ② 結果を統合
  ③ AIで「今日の優先事項TOP3」を生成
  ④ Webアプリに1画面表示

使う頻度:    毎朝7:00 自動実行 (定期実行のフラッグシップ)
実行時間:    5-10分
1実行原価:   $0.15-0.30
```

## 3. スキル定義JSONスキーマ

### 3.1 既存設計の拡張 (focusmap-pivot.md)

```json
{
  "id": "email-summary",
  "name": "メール要約",
  "description": "Gmail/Outlookの未読メールを3行で要約します",
  "icon": "📧",
  "category": "morning",

  "approval_type": "auto",           // auto / confirm / interactive
  "default_schedule": "0 7 * * 1-5", // 平日朝7時 (オプション)

  "required_auths": [                // 必要な認証
    {
      "service": "gmail",
      "scope": "Workspace共有 | 個人専用",
      "fallback": "outlook"
    }
  ],

  "configurable": {                  // ユーザーが設定可能なパラメータ
    "max_emails": {
      "type": "number",
      "default": 50,
      "min": 10,
      "max": 200,
      "label": "取得するメール数"
    },
    "lookback_hours": {
      "type": "number",
      "default": 24,
      "label": "過去◯時間のメール"
    },
    "summary_language": {
      "type": "enum",
      "options": ["ja", "en"],
      "default": "ja",
      "label": "要約の言語"
    }
  },

  "steps": [
    { "label": "メールサービスにログイン", "auto": true },
    { "label": "未読メール取得", "auto": true },
    { "label": "AI要約 + 重要度判定", "auto": true },
    { "label": "Webアプリに結果表示", "auto": true }
  ],

  "prompt_template": "scripts/email-summary を実行。lookback_hours={lookback_hours}, max_emails={max_emails}。日本語で3行要約、重要度を高/中/低で分類。",

  "estimated_cost_usd": 0.08,        // 1実行平均原価
  "estimated_duration_sec": 180,
  "model_recommendation": "gemini-flash"
}
```

### 3.2 スキーマの設計原則

- **configurable** で個別ユーザーの好みに合わせる (取得件数 / 言語 / 期間)
- **prompt_template** はテンプレ文字列、`{variable}` で configurable を注入
- **estimated_cost_usd** を表示してユーザーが実行前にコスト判断できる
- **required_auths** で必要な認証を明示、未認証ならスキル実行ボタンが無効化

## 5. テンプレマーケットプレイス (Phase 4以降構想)

### 5.1 想定

- 北村 / Focusmap社が公式テンプレを増やす
- ユーザーが自作スキルをテンプレ化して公開できる
- 評価・レビュー機構
- 一部テンプレは有料 (1回限り買い切り or 月額)

### 5.2 ガバナンス

- 公開前審査 (悪意あるテンプレを排除)
- 認証情報を盗むテンプレを構造的に作れない設計 (sandboxing)
- 利用規約違反テンプレの即時停止

### 5.3 MVP では入れない

理由:
- マーケットプレイス機構の実装が重い (審査 + 課金 + レビュー)
- 初期テンプレ7個で「便利」を訴求できれば十分
- ユーザーが増えてから後発で導入
