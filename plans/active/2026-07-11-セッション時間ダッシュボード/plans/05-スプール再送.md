# 05 Turso送信失敗のスプール再送

## 目的

A案決定（2026-07-11・md正本＋Tursoミラー維持）の唯一の弱点＝送信失敗時に追記式の履歴（session_events / session_logs）が恒久欠測になる問題を、ローカルスプール＆再送で「最終的に必ず一致（eventually consistent）」へ格上げする。

## 方針

1. `_turso_execute` が失敗（例外・タイムアウト・トークン欠落）した時、**events/logs系の文だけ**をJSONL（1行=1バッチ分の statements）で `hooks/session-board/state/turso-spool.jsonl` へ追記する。`at`/`created_at` は文の組立時に焼き込み済みなので、遅延再送でも時刻は正しい。**sessions の upsert/delete はスプールしない**（自己修復に任せる・古い現在値の追い越し上書きを防ぐ）。inbox宛（goal-add）はboardと別ファイルにするか宛先情報を行に含める。
2. 再送: board.py の毎実行の送信フェーズ末尾で、スプールが非空なら古い順に上限N文（例50）を1バッチで再送し、**成功時のみ該当行を削除**（tmp書き→原子的置換）。失敗したらそのまま残す。
3. `SESSION_BOARD_NO_TURSO` が立っている時はスプール追記も再送も一切しない（テスト分離）。
4. スプールは生成物として `.gitignore`（state/ 配下）。並行安全のため板と同じ flock パターンの専用ロックを使う。
5. README の送信説明へスプール仕様を追記し、決定ログとして「2026-07-11 一本化調査→A案維持・スプール導入・非同期化は見送り」を1行残す。

## 制約

- board.py は本番稼働中。編集ごとに `python3 -m py_compile` で構文確認し、壊れた状態を残さない。
- 行フォーマット・状態3値・旧形式互換を割らない。secret/トークン値を出力しない。
- コミットはパス指定（session-board配下のみ・`git add -A` 禁止）・push しない。
