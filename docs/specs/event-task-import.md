# 仕様書: カレンダーイベント自動取り込み

## 概要

Google カレンダーのイベントを自動的に tasks テーブルに取り込み、
全アイテムでタイマー・サブタスク・完了管理を統一的に使えるようにする。

---

## 操作ルール

### 自動取り込み

| 操作 | 動作 |
|------|------|
| 今日ビュー表示 | 7日先までのイベントを自動取り込み（初回は一括、以降は差分） |
| 日付変更 | 新しく範囲に入った1日分を追加取り込み |
| 手動リフレッシュ | 全範囲を強制再同期 |

### フィルタリング

| イベント種別 | 取り込み |
|-------------|---------|
| 通常イベント | ✅ 取り込む |
| 繰り返しイベント（各出現） | ✅ 取り込む |
| 共有カレンダーのイベント | ✅ 取り込む（表示ONのカレンダーのみ） |
| 全日イベント（誕生日・祝日） | ❌ 除外 |
| キャンセル済みイベント | ❌ 除外 |
| 非表示カレンダーのイベント | ❌ 除外 |

### Source of Truth（フィールドごとの正）

| フィールド | 正のデータ源 | 同期動作 |
|-----------|-------------|---------|
| タイトル | Google Calendar | Google→アプリ（上書き） |
| 開始時間 | Google Calendar | Google→アプリ（上書き）※アプリからの変更はGoogle APIで即反映済み |
| 終了時間 | Google Calendar | 同上 |
| カレンダーID | Google Calendar | Google→アプリ（上書き） |
| タイマー記録 | アプリ | 同期対象外（アプリのみで管理） |
| サブタスク | アプリ | 同期対象外（アプリのみで管理） |
| 完了状態 | アプリ | 同期対象外（アプリのみで管理） |
| stage | アプリ | 同期対象外（アプリのみで管理） |

### updated_at 保護

| 条件 | 動作 |
|------|------|
| タスクの updated_at が直近5分以内 | 同期スキップ（ユーザーが操作中の可能性） |
| 5分以上経過 | 通常通り同期 |

### 孤児タスク管理（ソフトデリート）

| 条件 | 動作 |
|------|------|
| Google側でイベント削除 | `deleted_at = now()` を設定（ソフトデリート） |
| Google側でイベント復元 | `deleted_at = null` に戻す（復活） |
| ソフトデリート後30日経過 | 完全削除（サブタスク含む） |

## フロー

### 初回取り込みフロー

```
今日ビュー表示
  ↓
useCalendarEvents で7日分取得（既存キャッシュ活用）
  ↓
フィルタリング（全日イベント除外、非表示カレンダー除外）
  ↓
既存の imported tasks を google_event_id で取得
  ↓
差分計算:
  Google にあり → tasks にない → INSERT（新規タスク作成）
  両方にある → fingerprint 比較 → 変更あれば UPDATE（タイトル・時間のみ）
  tasks にあり → Google にない → SOFT DELETE
  ↓
import_synced_at を更新
```

### 差分同期フロー（日次）

```
アプリ起動時
  ↓
import_synced_at をチェック
  ├─ 今日と同じ → スキップ
  └─ 昨日以前 → 差分同期開始
       ↓
     新しく範囲に入った日のイベントだけ処理
       ├─ 新規 → INSERT
       ├─ 変更 → UPDATE（fingerprint が異なる & updated_at > 5分前）
       └─ 削除 → SOFT DELETE
       ↓
     import_synced_at を更新
```

### イベント変更検出フロー

```
useCalendarEvents のキャッシュ更新（5分ごと）
  ↓
取り込み済みタスクの fingerprint と比較
  ├─ 一致 → 何もしない
  └─ 不一致 → タイトル・時間を更新（updated_at > 5分前のもののみ）
```

---

## 制約

1. **一方向同期**: Google Calendar → アプリ。アプリ側のタイトル変更はGoogleに反映されない（タイトルはGoogleが正）
2. **時間変更の既存同期**: アプリでD&Dで時間変更 → 既存の useTaskCalendarSync でGoogleに反映済み → 次の同期でGoogleから読んでも同じデータ
3. **全日イベント**: 取り込み対象外。タイムラインには表示するがタスク化しない
4. **取り込み範囲外**: 7日先を超えるイベントは取り込まない。範囲は設定で変更可能
5. **ソフトデリート30日**: 完全削除後はデータ復元不可

---

## エッジケース

| ケース | 対応 |
|--------|------|
| 繰り返しイベントの1回だけ変更 | Google API が個別出現として返すため、その出現のタスクだけ更新 |
| 繰り返しイベント全体の削除 | 全出現がGoogle APIから消えるため、全タスクがソフトデリート |
| カレンダーの表示をOFFに変更 | 次の同期でそのカレンダーのイベントはフェッチされない → ソフトデリート |
| 同じイベントが重複取り込み | google_event_id をユニークキーとして使用 → UPSERT で防止 |
| ネットワークエラーで同期失敗 | 次回起動時に再試行。import_synced_at は更新されないため再実行 |
| 取り込み済みタスクを手動削除 | source='google_event' のタスクは手動削除可能。次の同期で再作成される |
| event_completions に既存データあり | Phase 2 で tasks.status に移行後、テーブル削除 |
