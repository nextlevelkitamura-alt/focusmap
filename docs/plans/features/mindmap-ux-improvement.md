# 設計プラン: マインドマップ操作感改善（XMind風）

## 目標
XMindのようなスムーズで直感的なマインドマップ操作を実現する。
キーボードだけで高速にタスクの作成・編集・削除・移動ができる操作体系。

## 現状の問題点

### P1: Enter キーで兄弟タスクが生成されない
- **現状**: ノード選択 → Enter で「確定モード」に移行するだけ（3段階モデル）
- **期待**: XMind風 — Enter で即座に兄弟タスク（同階層）を生成、フォーカスは新ノードへ
- **影響箇所**: `mind-map.tsx` の `handleKeyDown` イベントハンドラ

### P2: Delete キーで連続削除できない
- **現状**: ノード選択 → Delete で削除はされるが、次のノードが自動選択されない
- **期待**: 削除後に次の兄弟（下→上→親の順）にフォーカスが移動し、連続 Delete が可能
- **選択順序（削除後の自動選択）**:
  1. 次の兄弟（下のノード）
  2. 前の兄弟（上のノード）
  3. 親ノード
  4. なければ選択解除
- **影響箇所**: `mind-map.tsx` の delete ハンドラ + ノード選択ロジック

### P3: 編集モードの操作が直感的でない
- **現状**: 3段階モデル（選択 → F2/Space/文字入力で編集 → Enter で確定 → Enter でさらに兄弟追加）
- **期待**: XMind風 2段階モデル
  - **選択モード**: ノードをクリック or 矢印キーで選択（青枠表示）
  - **編集モード**:
    - 文字キーを押す → 既存テキストを**上書き**して編集開始
    - F2 / ダブルクリック → 既存テキストの**末尾にカーソル**で編集開始
    - マウスでテキスト上をクリック → その位置にカーソル（テキスト追記可能）
  - **Enter で確定** → 選択モードに戻る（兄弟タスクは生成しない）
- **影響箇所**: `mind-map.tsx` の TaskNode, GroupNode の編集ロジック全体

### P4: その他 XMind風の改善（将来）
- Tab で子タスク追加（既存実装 → 維持）
- Ctrl+Enter で兄弟追加（Enter と同じ動作にする案も検討）
- 複数選択→一括削除のフォーカス管理
- ノードのドラッグ&ドロップによる並べ替え（グループ間移動）
- Undo/Redo 対応

## 操作キーマップ（改善後）

| キー | 選択モード | 編集モード |
|------|-----------|-----------|
| Enter | 兄弟タスクを追加 | テキスト確定 → 選択モードに戻る |
| Tab | 子タスクを追加 | — (ブラウザデフォルト) |
| Delete/Backspace | ノード削除 → 次のノードに自動移動 | テキスト編集 |
| F2 | 編集モード開始（末尾カーソル） | — |
| Space | 編集モード開始（末尾カーソル） | スペース入力 |
| 文字キー | 編集モード開始（上書き） | テキスト入力 |
| ↑↓←→ | ノード間移動 | テキスト内カーソル移動 |
| Escape | 選択解除 | 編集キャンセル → 選択モードに戻る |
| ダブルクリック | 編集モード開始（末尾カーソル） | — |
| クリック | ノード選択 | クリック位置にカーソル |
