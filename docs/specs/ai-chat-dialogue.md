# AIチャット型対話 仕様書

> メモ機能 Phase 5: エージェンティックなAI提案 + フローティングチャット

## 概要

メモ保存時にAIが自動分析し、**具体的な選択肢付きのインライン提案**をメモカード内に表示。
ユーザーはボタンタップだけで操作を完了できる。手動でフローティングチャットを開いて自由対話も可能。

## 2つのフロー

### 1. インライン提案（メイン・自動）
- **トリガー**: メモ保存時に自動
- **表示場所**: メモカード内（AI分析結果エリア）
- **UX**: 選択肢ボタンで対話的に操作（テキスト入力不要）
- **マルチステップ**: 場所選択 → 確認 → 実行の段階的フロー

### 2. フローティングチャット（サブ・手動）
- **トリガー**: 画面右下のアイコンタップ
- **表示場所**: フローティングパネル
- **UX**: テキスト/音声入力 + 選択肢ボタン併用
- **7ラリー制限**: セッション要約でコスト管理

## インライン提案フロー

### Step 1: メモ保存 → 自動分析
```
メモ保存
  ↓ 自動で /api/ai/analyze-memo 呼び出し
分析中... (ローディング表示)
  ↓
結果表示 (メモカード内)
```

### Step 2: 提案表示（classification=map の場合）
```
🗺️ マップに追加
「○○プロジェクト > □□タスク」の下に追加しますか？

[追加する] [別の場所を選ぶ] [スキップ]
```

### Step 3a: 「追加する」→ 実行
```
✅ タスク「{メモ内容}」を追加しました
```

### Step 3b: 「別の場所を選ぶ」→ プロジェクト選択
```
どのプロジェクトに追加しますか？

[プロジェクトA] [プロジェクトB] [プロジェクトC]
```

### Step 4: プロジェクト選択後 → ノード選択
```
どこに追加しますか？

[ルートに追加] [タスクX の下] [タスクY の下]
```

### Step 5: 実行
```
✅ 「プロジェクトA > タスクX」の下に追加しました
```

### classification=calendar の場合
```
📅 カレンダーに追加
「{メモ内容}」を 2026-02-25 (火) 10:00 に追加しますか？

[追加する] [日時を変更] [スキップ]
```

## フローティングチャット仕様（従来通り）

### UI
- 位置: 画面右下（モバイル: 50vh / PC: 400x500px）
- テキスト入力 + 音声入力 + 選択肢ボタン
- 7ラリー制限 + セッション要約

### AIの応答にも選択肢を含む
```
ユーザー: 「来週の企画会議を追加したい」
AI: 「カレンダーに追加します。日時はいつですか？」
    [月曜 10:00] [火曜 10:00] [水曜 10:00] [自分で入力]
```

## 操作ルール

| 操作 | トリガー | 実行方法 |
|------|---------|---------|
| マップにタスク追加 | 自動提案 or チャット | POST /api/tasks |
| カレンダーに予定追加 | 自動提案 or チャット | POST /api/tasks + sync |
| メモのプロジェクト紐付け | 自動提案 or チャット | PATCH /api/notes |
| メモのアーカイブ | 追加完了後に自動 | PATCH /api/notes |

### 制約
- 削除はAI実行不可
- 追加操作は確認ボタン必須
- マルチステップで段階的に確認

## 実装フェーズ

### Phase 5a: インライン提案（メイン）
1. メモ保存時の自動AI分析
2. 分析結果からインライン提案UI生成
3. マルチステップ選択フロー（プロジェクト→ノード）
4. アクション実行 + 結果表示

### Phase 5b: フローティングチャット改善
1. AIの応答に選択肢ボタンを含める
2. 音声入力統合
3. 選択肢+テキスト入力のハイブリッド

### Phase 5c: セッション管理
1. ai_chat_summaries テーブル
2. 要約生成API
3. セッション開始時の要約読み込み
