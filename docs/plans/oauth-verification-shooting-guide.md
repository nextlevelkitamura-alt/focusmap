# 撮影マニュアル（素人向け・Mac前提）

このファイルは「動画編集も画面録画もやったことない人」が、このファイルだけ見て撮影を完了できるように書いています。専門用語は最小限。1ステップずつチェックしながら進めてください。

> **必読**: 撮影の前に [oauth-verification.md](./oauth-verification.md) の §1 と §8 を読んでください。本ファイルは「実際に画面の前に座った状態で何を押すか」だけを書いています。

---

## 0. 用意するもの

- Mac本体（推奨: 13インチ以上）
- マウス or トラックパッド
- 外部ディスプレイは **使わない**（解像度がバラつくので内蔵だけで撮る）
- Chrome（普段使ってないプロファイルを新規で1つ作る）
- 撮影アカウント用のGmail（普段使いのアカウントは使わない）

## 1. 撮影前のMac側準備（所要15分）

### 1-1. 通知を全部止める

撮影中にSlack通知やメールバナーが出ると映り込んでアウト。

1. 画面右上の **コントロールセンター**（時計の左にあるスイッチアイコン）をクリック
2. **「集中モード」** → **「おやすみモード」をON**

### 1-2. メニューバーを最小化

1. Macの **システム設定** → **コントロールセンター**
2. メニューバー上にある余計なアイコン（不要なアプリ）を **「メニューバーに表示しない」** に設定
3. 時計・Wi-Fi・電源・コントロールセンターだけ残ればOK

### 1-3. デスクトップをきれいに

1. デスクトップに散らかってるファイル/フォルダを `~/Documents/tmp/` あたりに一時退避
2. デスクトップ壁紙はシンプルなものに（黒・白・無地推奨）

### 1-4. Chromeの撮影用プロファイルを作る

1. Chrome右上のプロフィールアイコン → **「追加」**
2. プロファイル名: `Focusmap OAuth撮影用`
3. 新しいプロファイルが開いたら:
   - **言語をEnglishに**:
     1. アドレスバーに `chrome://settings/languages` を貼って開く
     2. **English (United States)** を追加して **一番上に移動**
     3. 「このGoogle Chromeで使用する」ボタンを押す（出てきたら）
     4. **Chromeを完全に終了→再起動**
     5. メニューが英語になっていればOK（File / Edit / View ...）
   - **ブックマークバーを非表示**: `View` → `Always Show Bookmarks Bar` を **OFF**
   - **拡張機能アイコンを非表示**: パズルピース型の拡張機能アイコンは右クリックで非表示に

### 1-5. テスト用Gmailアカウントを準備

- Focusmap撮影専用のGoogleアカウントを1つ用意（既存でも新規でもOK）
- 過去にFocusmapと連携した経歴があれば https://myaccount.google.com/permissions で **Focusmapを削除**（初回同意画面を撮るため）

### 1-5.5. 古いCookieを必ず削除（PKCEエラー回避）

ドメイン切替（旧 `*.run.app` → `focusmap-official.com`）の直後は、ブラウザに古いCookieが残っていて **`error=exchange&reason=PKCE+code+verifier+not+found+in+storage`** というエラーが出る既知の問題があります。撮影前に必ず以下を実行してください。

1. 撮影用Chromeプロファイルで `focusmap-official.com` を開く
2. アドレスバー左の鍵アイコン → **「Cookieとサイトデータ」** → **すべて削除**
3. 同様に **shikumika-app-364jgme3ja-an.a.run.app** のCookieも残っていれば削除（履歴→Cookieと他のサイトデータから検索）
4. ブラウザを **完全終了→再起動**
5. これで初回サインインから綺麗にやり直せる

> もし撮影中にこのエラーが出たら、 **そのステージだけ撮り直し**＋上記Cookie削除を再実行。コード側にも修正の余地はあるが、撮影だけならCookie掃除で回避できる。

### 1-6. Chromeウィンドウサイズを1280x720に固定

> なぜ1280x720？ → 1080p（1920x1080）で録画したいけど、Retinaディスプレイは見かけの解像度の2倍で記録される。なのでChromeウィンドウを **1280x720** にして撮ると、最終的に2560x1440 → 1920x1080にダウンスケールできて綺麗。

1. Chromeのアドレスバーに以下を貼り付けてEnter（一発でリサイズできる）:
   ```
   javascript:window.resizeTo(1280,720)
   ```
   ※ 最近のChromeはこのJavaScript URL貼り付けがブロックされる場合あり。その場合は次の方法:

2. **手動でやる場合**:
   - Chromeウィンドウを左上に寄せる
   - 右下の角をドラッグして、画面端まで広げない（ある程度小さく）
   - 完璧でなくてもOK。後でRemotion側でクロップできる

## 2. 録画ソフトの準備（QuickTime使用）

Macに最初から入っているQuickTime Playerで十分。

### 2-1. QuickTimeを起動

1. **Launchpad** → **QuickTime Player**
2. メニューバーの **File** → **New Screen Recording** （または `Cmd + Ctrl + N`）
3. 録画ツールバーが画面下に出る

### 2-2. 録画範囲を設定

- ツールバーの真ん中の左から3番目（点線の四角アイコン）= **選択範囲を録画**
- ドラッグでChromeウィンドウだけを囲む（アドレスバーから下端まで全部含める）
- ツールバー右側の **Options** をクリック:
  - **Microphone: None**（音声は録らない）
  - **Show Mouse Clicks in Recording: ON**（クリック位置に丸が出て見やすくなる）

### 2-3. 録画開始/停止

- **Record** ボタンで録画開始
- 停止は画面右上メニューバーの **■（停止アイコン）** をクリック
- 自動で `~/Desktop` に保存される

### 2-4. 1ステージずつ別ファイルで録画する

**重要**: 全部を1本で撮ろうとせず、**ステージごとに別ファイル**で撮る。失敗したらそのステージだけ撮り直せばOK。

### ステージD: OAuth同意画面【最重要・神経使う】

**目的**: 申請したscope（calendar.events / calendar.calendarlist.readonly）と同意画面表記が一致していることを見せる
**目安時間**: 20秒くらい

1. ダッシュボードに着地している状態から
2. QuickTime録画開始
3. **カレンダー設定** を開く（サイドバーの設定 or 設定ページ）
4. **「連携する」ボタン** が見える状態で1秒静止
5. **「連携する」をクリック**
6. Google OAuth同意画面が表示される
7. **アドレスバーをマウスでクリック**して `client_id=` の文字が見える状態にして1秒静止 ★ここ必須
8. 同意画面に戻って、 **「Focusmap wants access...」** のテキストが見えるところで1秒静止
9. **下にスクロール**して **2つの権限が両方見える状態** にする:
   - "See, edit, share, and permanently delete all the calendars you can access using Google Calendar"
   - "See the list of Google calendars you're subscribed to"
10. その状態で3秒静止 ★ここ必須
11. さらに下の **Privacy Policy / Terms of Service リンク** が見える位置で2秒静止
12. **Continue（または「許可」「Allow」）ボタンをクリック**
13. Focusmapに戻ってくるまで撮る
14. 録画停止
15. ファイル名 **`stage-D-consent.mp4`** にしてraw/に移動

> **撮り直し判定**:
> - 同意画面が **日本語** で出た → Chrome言語設定を見直して撮り直し
> - 同意画面に **shikumika** など旧名が出た → OAuth Consent Screenのapp nameを `Focusmap` に修正してから撮り直し
> - 2つのscope両方が画面に映ってない → スクロールしてもう一度撮影

### ステージG: calendar.events WRITE（書き込み）【最重要】

**目的**: Focusmapからカレンダーに「予定を作る・編集する・削除する」ことを見せる
**目安時間**: 30〜40秒くらい

> このステージは長いので、 **CREATE / UPDATE / DELETE で別ファイルでもOK**。その場合は `stage-G1-create.mp4` `stage-G2-update.mp4` `stage-G3-delete.mp4` のように分けて保存し、私に伝えてください。

#### G-1. 作成（CREATE）
1. Focusmap内で **タスクを1件作成** → 「カレンダーに追加」アクション
2. 別タブで `https://calendar.google.com` を開く（**同じテストアカウントでログイン済みの状態にしておく**）
3. 作成された予定が **Googleカレンダーに反映されている** ことを見せる → 2秒静止

#### G-2. 更新（UPDATE）
1. Focusmapに戻る
2. 同じ予定を **編集**（タイトル変更 or 時間変更）
3. Googleカレンダーのタブに切り替えて、 **変更が反映されている** ことを見せる → 2秒静止

#### G-3. 削除（DELETE）
1. Focusmapに戻る
2. 同じ予定を **削除**
3. Googleカレンダーのタブで **予定が消えている** ことを見せる → 2秒静止

ファイル名 **`stage-G-events-write.mp4`** （または分割版）にしてraw/に移動

## 5. よくあるトラブル

| 症状 | 対処 |
|------|------|
| QuickTimeのファイルが `.mov` で出る | そのままraw/に置いてOK（拡張子だけ `.mp4` にリネームでも可）。Remotion側で読める |
| 同意画面が日本語 | Chromeの言語設定を見直して、Chrome完全再起動 |
| 「アプリは確認されていません」警告が出る | 「詳細」→「{アプリ名}（安全ではないページ）に移動」で進む。これも撮影に含めてOK（審査前の状態として正常） |
| 連携済みになっていて同意画面が出ない | https://myaccount.google.com/permissions でFocusmapを削除してから再度撮影 |
| 撮影中に通知が映り込んだ | そのステージだけ撮り直し（おやすみモードを確認） |
| カレンダーの予定が私物 | テスト用アカウントで撮り直し |
