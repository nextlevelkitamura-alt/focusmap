# shikumika AI コア機能 企画書

## 日付
2026-02-27

## コンセプト
**「対話から始まり、マインドマップに落ちる。行動まで繋がる。」**

今のAIチャットは「聞いたら答えが返ってくるだけ」。
shikumika のAIは、対話しながら考えを構造化し、行動まで導くエージェント。

## ユーザー体験（具体例）

### シーン1: 壁打ち → マインドマップ化
```
ユーザー: 「転職するか迷ってるんだけど」
AI: 「今の仕事の不満と、理想の状態を聞かせてもらえますか？」
ユーザー: 「給料が低い。でも仕事内容は好き。成長もできてる」
AI: （対話しながら自動でマインドマップを生成）

  転職の検討
  ├─ 現状の良い点
  │  ├─ 仕事内容が好き
  │  └─ 成長できている
  ├─ 現状の課題
  │  └─ 給料が低い
  ├─ 選択肢
  │  ├─ 今の会社で昇給交渉
  │  ├─ 副業で収入補填
  │  └─ 転職活動を始める
  └─ 次のアクション
     └─ （AIが提案）

AI: 「整理するとこんな感じですね。どの選択肢を深掘りしますか？」
AI: 「ちなみに、来週の空いてる時間に情報収集の時間を入れましょうか？」
→ Google Calendar に予定を追加
```

### シーン2: ノード内のメモ
マインドマップの各ノードをクリックすると：
- 対話の中で出た詳細メモ
- 関連する数字やデータ
- AIが追加した補足情報
- 参考リンクや資料

### シーン3: スキルの活用
AIが文脈に応じてスキルを使い分ける：
- 📅 カレンダー: 「この日に予定入れましょうか」
- ✅ タスク追加: 「やることリストに入れておきますね」
- 🔍 リサーチ: 「関連情報を調べてみますね」
- 📊 分析: 「今月の進捗を見てみましょう」

## AIに求める能力

### 必須
1. **壁打ち・深い対話**: 表面的な回答ではなく、掘り下げる質問ができる
2. **構造化**: 対話内容を自動でマインドマップノードに変換
3. **文脈理解**: ユーザーの過去の会話・コンテキストを踏まえた応答
4. **行動提案**: 「じゃあこうしましょう」と具体的な次のステップを提案
5. **スキル実行**: カレンダー登録、タスク追加などを実際に実行

### 差別化ポイント
- ChatGPT: 答えを返すだけ（行動に繋がらない）
- Notion AI: ドキュメント内で完結（構造化・行動連携が弱い）
- shikumika: 対話 → 構造化 → 行動 が一気通貫

## AI アーキテクチャ: モデル非依存 + スキル連動自動切替

### 設計思想
- **特定のAIプロバイダーに依存しない**
- スキル（tool）の定義は共通。接続先のAIだけ差し替え可能
- スキルの種類に応じて、最適なモデルを自動選択する

### スキル × モデル 自動切替マッピング

| スキルの種類 | 求められる能力 | 推奨モデル | コスト帯 |
|---|---|---|---|
| 📅 カレンダー追加 | 正確なパラメータ抽出 | GPT-4o-mini | 最安 |
| ✅ タスク追加 | 正確なパラメータ抽出 | GPT-4o-mini | 最安 |
| 🏋️ 習慣記録 | 単純な記録処理 | GPT-4o-mini | 最安 |
| 🗺️ マップ構造化 | 構造化・分類能力 | GPT-4o / Claude Sonnet | 中 |
| 💬 壁打ち・深い対話 | 論理的思考・共感・深掘り | Claude Sonnet / GPT-4o | 中〜高 |
| 🔍 リサーチ | 情報整理・要約 | GPT-4o / Claude Sonnet | 中 |
| 📊 分析・設計 | 高度な論理・構造設計 | Claude Sonnet / Opus | 高 |

### 切替の仕組み（2段階）

```
ユーザーの発言
    ↓
[Stage 1] ルーター（GPT-4o-mini / 最安モデル）
    「この発言はどのスキルに該当するか？」を判定
    ↓
[Stage 2] スキルに紐づくモデルに自動ルーティング
    軽いスキル → GPT-4o-mini がそのまま処理
    重いスキル → Claude Sonnet / GPT-4o に引き継ぎ
```

### 切替テーブル（設定で変更可能）
```typescript
const SKILL_MODEL_MAP = {
  'calendar':      { provider: 'openai',    model: 'gpt-4o-mini' },
  'task-add':      { provider: 'openai',    model: 'gpt-4o-mini' },
  'habit':         { provider: 'openai',    model: 'gpt-4o-mini' },
  'mindmap':       { provider: 'anthropic', model: 'claude-sonnet-4-6' },
  'brainstorm':    { provider: 'anthropic', model: 'claude-sonnet-4-6' },
  'research':      { provider: 'openai',    model: 'gpt-4o' },
  'analysis':      { provider: 'anthropic', model: 'claude-sonnet-4-6' },
}
// ↑ 新モデルが出たら、ここを変えるだけで全体に反映
```

### プロバイダー抽象化
```typescript
// 共通インターフェース（全プロバイダー共通）
interface AIProvider {
  chat(messages: Message[], tools: Tool[]): Promise<Response>
}

// 実装は各プロバイダーごと
class OpenAIProvider implements AIProvider { ... }
class AnthropicProvider implements AIProvider { ... }
class GoogleProvider implements AIProvider { ... }  // 将来追加も容易
```

### コスト試算（最適化後）
- 1ユーザー/日: 平均5回のAI利用
  - 軽い処理（mini）3回: 約 $0.002
  - 深い処理（Sonnet/4o）2回: 約 $0.05
- 月間コスト/ユーザー: 約 $0.50-1.50（¥75-220）
- **¥980 プランで粗利率 75-90%**

### なぜモデル非依存か
- AI業界は半年で勢力図が変わる。特定ベンダーに縛られるとリスク
- 新モデルが出たら設定テーブルを変えるだけで即対応
- コスト最適化が柔軟にできる（安いモデルが出たらすぐ切替）
- ユーザーには裏側のモデルは見えない。体験だけが良くなる

## スキルシステム設計

### コンセプト: 「アプリストア」のようなスキル管理
ユーザーがスキルを選んでインストール/有効化できる。

### スキル一覧（初期）
| アイコン | スキル名 | 説明 | 外部連携 |
|---|---|---|---|
| 📅 | カレンダー | 予定の確認・追加・変更 | Google Calendar API |
| ✅ | タスク管理 | マインドマップにタスクを追加・完了管理 | 内部DB |
| 💬 | 壁打ち | 深い対話・思考の整理 | なし（AI単体） |
| 🗺️ | マップ構造化 | 会話をマインドマップに自動変換 | 内部DB |

### スキル一覧（将来）
| アイコン | スキル名 | 説明 | 外部連携 |
|---|---|---|---|
| 📧 | メール | メールの要約・返信下書き | Gmail API |
| 📝 | メモ | 長文メモの作成・要約 | なし |
| 🔍 | リサーチ | Web検索して情報収集 | Search API |
| 📊 | レポート | 進捗レポートの自動生成 | 内部DB |
| 🏋️ | 習慣 | 習慣の記録・分析 | 内部DB |
| 💰 | 家計 | 支出管理・予算アドバイス | Money Forward API等 |

### スキルのUI
- 設定画面にスキル一覧（アイコン + 名前 + 説明）
- ON/OFF トグルでインストール/アンインストール
- 各スキルに必要な外部連携の認証フロー
- AIが会話中に使えるスキルを自動判別

## 優先順位（実装順序）

### Step 1: AI基盤のモデル非依存化
- AIProvider 抽象レイヤーを実装（OpenAI / Anthropic / Google 共通インターフェース）
- 既存の3スキル（schedule, task-add, chat）を共通 tool use 形式に移行
- スキル×モデル自動切替テーブルの実装
- ルーター（GPT-4o-mini）でスキル判定 → 最適モデルに自動ルーティング

### Step 2: 壁打ち対話の強化
- 深掘り質問ができるプロンプト設計
- ユーザーコンテキストの活用強化
- 対話品質の向上

### Step 3: 対話→マインドマップ自動変換
- 会話内容をリアルタイムでノード化
- ノード内に詳細メモを格納
- マップの自動レイアウト最適化

### Step 4: スキルシステムのUI
- スキル管理画面（アイコン付き一覧）
- スキルのON/OFF機能
- スキル追加時の認証フロー

### Step 5: 外部連携スキルの拡充
- メール、リサーチ、レポートなど順次追加
