FOCUSMAP · REPO-LOCAL LOOP GUIDE

Focusmapの新規loopは
loops/ に置く

ここはFocusmapが所有する新規loopの実体正本です。基盤は発見の入口、launchdは実行時刻、registryは全体状態を担当します。AIが読む正本は同階層の AGENTS.md です。

役割を分ける

実装の正本

この loops/

新規loopは loops/<loop-id>/ に置きます。基盤へ同じ実装をコピーしません。

実行の正本

plist / launchd

いつ動くかはplistと実機の launchctl が決めます。基盤リンクは実行経路ではありません。

発見の入口

基盤directory link

implementation-links/Focusmap はこのroot全体へ繋がり、人が基盤から辿るためだけにあります。

最小構成

必須は定義だけ。必要な実装・ログ・launchdだけを後から足します。

Focusmap/loops/
├── AGENTS.md                 AI向けの正本
├── CLAUDE.md → AGENTS.md
├── AGENTS.html               人間向けの説明
└── <loop-id>/
    ├── loop.md               目的・発火・runner・停止手順
    ├── scripts/              実装が必要な時だけ
    ├── logs/                 ファイルログが必要な時だけ・gitignore
    └── *.plist               launchdを使う時だけ

新設の流れ

  1. Global Skill loop-creator で所有repoとこのrootを確定する。
  2. loop.md から作り、必要性がある時だけscript・logs・plistを追加する。
  3. 基盤のroot directory linkと、その時点のregistry正本を照合する。
  4. launchdの有効化・再登録・周期変更は、対象とrollbackを確認した人の承認後だけに行う。
既存導線は据え置きです。
このrootを作っただけでは、scripts/focusmap-agent/、scripts/task-runner.ts、既存のscriptsやlaunchdを移動・変更しません。

判断するときの正本

基盤のloop規則: AIエージェント基盤/loops-registry/AGENTS.md

リンクの安全規則: loops-registry/implementation-links/AGENTS.md
