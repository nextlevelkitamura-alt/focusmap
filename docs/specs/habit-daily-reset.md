# 仕様書: 習慣機能改善（日次完了リセット・TODOビュー統合・週間達成率UI）

## 概要

習慣の子タスク完了状態を日付ごとに管理し、毎日自動リセットする機能。
今日ビューにハイブリッド表示（横スクロール ↔ カード展開）を導入し、
週間ビューにヒートマップグリッドで達成率を可視化する。

---

## 操作ルール

### 子タスク日次完了

| 操作 | 動作 | DB変更 |
|------|------|--------|
| 子タスクをチェック | 今日の日付で完了記録 | `habit_task_completions` に INSERT |
| 子タスクのチェック解除 | 今日の完了記録を削除 | `habit_task_completions` から DELETE |
| 全子タスクが完了 | 習慣全体も自動完了 | `habit_completions` にも INSERT |
| 全子タスク完了後に1つ解除 | 習慣全体の完了も取消 | `habit_completions` から DELETE |
| 日付が変わる | 全チェックがリセット | DB操作なし（レコードが存在しないだけ） |

### 習慣全体の完了（子タスクなしの場合）

| 操作 | 動作 |
|------|------|
| 習慣をチェック | `habit_completions` に今日の日付で記録（従来と同じ） |
| 習慣のチェック解除 | `habit_completions` から今日の記録を削除（従来と同じ） |

### 今日ビュー: ハイブリッド表示

| 状態 | 表示 |
|------|------|
| 折りたたみ（デフォルト） | 横スクロールバー: アイコン + 進捗(N/M) + ストリーク + ▼ボタン |
| 展開 | カード形式: ヘッダー(達成率プログレスバー) + 各習慣カード(子タスクチェック付き) + ▲ボタン |

### 週間ビュー: ヒートマップグリッド

| 操作 | 動作 |
|------|------|
| グリッド表示 | 縦=習慣名、横=月〜日。セル色で達成率表示 |
| 前週/次週 | ← → ボタンで週を切り替え |
| セルタップ | 該当日の詳細（完了した子タスク名）をツールチップ表示 |
| 達成率計算 | 完了子タスク数 ÷ 全子タスク数。子タスクなし: 完了=100%, 未完了=0% |

## フロー

### 子タスクチェックのフロー

```
1. ユーザーが子タスクをタップ
2. UIが即座に✓表示（Optimistic Update）
3. API呼び出し: POST /api/habits/task-completions
   { habit_id, task_id, completed_date: "today" }
4. 成功: 確定
5. 失敗: ✓を戻す + エラー表示
6. 全子タスク完了チェック
   → 全完了なら habit_completions にも自動記録
```

### 日付変更のフロー（リセット）

```
1. ユーザーがアプリを開く
2. useHabits が today の日付で完了記録を取得
3. 今日の完了記録がない子タスク = 未チェック表示
4. → 自動的にリセットされて見える（DBには前日の記録が残っている）
```

### ハイブリッド表示のフロー

```
1. 今日ビューを開く
2. デフォルト: 横スクロールバー表示（現在と同じ）
3. ▼ボタンをタップ
4. アニメーションでカード形式に展開
5. 子タスクのチェック操作が可能
6. ▲ボタンで折りたたみに戻る
```

## 達成率の色定義（ヒートマップ）

| 達成率 | 色 | Tailwind クラス |
|--------|-----|----------------|
| 0% | グレー | `bg-gray-200 dark:bg-gray-700` |
| 1-49% | 薄い緑 | `bg-green-200 dark:bg-green-900` |
| 50-99% | 中間の緑 | `bg-green-400 dark:bg-green-700` |
| 100% | 濃い緑 | `bg-green-600 dark:bg-green-500` |
| 非該当日 | 透明 | `bg-transparent` (習慣の曜日設定外) |

---

## 制約

1. `habit_task_completions` は同じ `task_id` + `completed_date` で最大1件（UNIQUE制約）
2. 過去の日付の完了記録は変更不可（UIでは今日分のみ操作可能）
3. 習慣の有効期間外（`habit_start_date`〜`habit_end_date`）はチェック不可
4. ヒートマップは最大過去4週間分まで表示
5. 展開/折りたたみの状態はメモリ上のみ（永続化しない）

---

## エッジケース

| ケース | 挙動 |
|--------|------|
| 子タスクが0個の習慣 | 習慣自体のチェックで完了/未完了を管理（従来通り） |
| 日付変更線をまたいでアプリ使用中 | 次回のデータ取得時にリセットされる（リアルタイム検知不要） |
| 子タスクを途中で追加 | 追加した日からカウント開始。過去日への影響なし |
| 子タスクを削除 | CASCADE で `habit_task_completions` の該当レコードも削除 |
| 習慣自体を削除 | CASCADE で全関連レコード削除 |
| 過去の完了記録がある日にタイムゾーンが変わる | `completed_date` はDATE型（時刻なし）のため影響なし |
| 週間ビューで習慣の曜日設定外の日 | セルを透明/ドット表示で「非該当」を明示 |
